Behavioural Design

This section of the Style Guide ensures users are supported through predictable interaction, usability, and feedback.

It includes UI behaviours such as hover states, form validation, button feedback, and accessible navigation ‚Äî key elements in the behavioural level of design.

Button Styles

Buttons follow consistent styling to signal interactivity. Hover, focus, and disabled states provide visual feedback.

Primary Button

High-contrast primary action. Hover and pressed states provide feedback. Disabled style signals non-interactive state.

Secondary Button

Alternative action with subdued colours. Hover/pressed states reinforce usability without disrupting hierarchy.

Icon Button

Icon-only button with aria-label for screen reader support. Suitable for utility actions.

Form Example

This form includes real-time validation and provides user feedback on both field errors and submission status.

Navigation and Feedback

Navigation menus use visual hierarchy and semantic HTML to support clarity. Menus highlight the active page, and feedback is offered via focus indicators and hover transitions.

Micro-Interactions

Hover effects on buttons and images give users cues about interactivity. These small interactions build trust and improve usability.

Accessibility Principles

  • All text meets minimum WCAG 2.2 contrast guidelines.
  • Navigation is keyboard accessible, and buttons are clearly labeled.
  • Images contain meaningful alt text and avoid decorative clutter.
  • Forms include validation feedback and labels to support screen readers.
